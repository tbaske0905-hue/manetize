AI UI Review

AIが作った画面を、
デザインで
本物にする。

Claude Code・Cursor・Codex で爆速プロトタイプを作れるようになった。

でも、画面の見た目がAIらしさを主張してしまう。

fix-ai.design は、スクリーンショットを送るだけで「なぜAIっぽく見えるか」を座標付きで返す UI レビューサービスです。

依頼を作成する 仕組みを見る
fix-ai review ./lp-hero.png
$ fix-ai review ./lp-hero.png

対応ツール Claude Code Cursor Codex Gemini CLI Cline

AIが作る画面には、
共通のパターンがある。

01

余白が均一すぎる

すべてのセクションに同じ padding。情報の重みが伝わらず、視線が止まらない画面になる。

02

フォントに強弱がない

見出し・本文・注釈のサイズ差が小さく、どこから読めばいいかが分からない。読み流される。

03

CTA が背景に沈む

ボタンの色がサイト全体の色調に同化している。最も重要な行動喚起が、目に入らない。

3ステップで、
AIへの修正指示が届く。

デザインの知識は不要。画面を送るだけで、Claude Code や Cursor に渡せる形で返します。

STEP 01

画面を送る

スクリーンショットまたは URL を送信します。LP・Webアプリ・管理画面、どの画面でも対応します。

STEP 02

AIっぽさを特定する

座標付きのコメントで問題箇所を明示します。「なぜ変えるか」の理由も一緒に返すので、AIへの指示精度が上がります。

STEP 03

そのままAIに渡す

Markdown / JSON / コード形式で納品します。Claude Code や Cursor に貼り付けるだけで修正が始まります。

実際の変化

Before — AI生成のまま

均一な余白 / フォントに強弱なし / CTAが目立たない

After — fix-ai 適用後

情報の重みが整理され、視線が自然に流れる
